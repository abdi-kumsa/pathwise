01  —  Project Overview

What I Am Building

This project involves designing and delivering a full-stack predictive analytics platform built around two interconnected intelligence dashboards. The platform draws from multiple data sources — institutional enrollment records, labor market feeds, regional economic indicators, and demographic data — and processes them through a multi-model machine learning pipeline to generate actionable forecasts and recommendations.

In plain terms: the platform answers two questions simultaneously — what does the job market actually need right now? and how should an institution adjust who it enrolls and what it teaches to match that? — backed by real data and predictive models, not assumptions.

The architecture is built for scale and maintainability: a cloud-ready data pipeline, a REST API backend serving ML inference in real time, and a responsive React frontend that makes complex data legible to both administrators and students. The two modules are designed to complement each other within a single unified application.

02  —  Module Breakdown

Two Modules, One Integrated Platform

Each module has a distinct audience and purpose, but they share the same data pipeline and ML backend — which keeps the system cohesive and reduces overhead significantly.

01
Market Demand Forecast Dashboard
For institutional leaders, department heads, and curriculum planners
Hiring & Market Intelligence
  • Top Hiring Companies & Roles — Bar charts and tree maps surfacing the organisations recruiting most actively, and which job titles are in highest demand
  • Regional Opportunity Maps — Geographic visualisations showing job and internship density by location, identifying high-demand regions
  • Recruitment Channel Analysis — Breakdown of which sourcing platforms are most active by industry or department
  • Job-to-Internship Ratio — Comparative charts showing the balance between full-time offers and experiential opportunities in the current market
Skill Gap & Curriculum Alignment
  • Skill Demand vs. Training Supply — Interactive mapping of what the market requires (AI, Big Data, Digital Marketing) against what the institution currently teaches
  • Graduate-to-Role Fit Classification — Categorises placement outcomes as Matched, Overeducated, or Undereducated to identify structural misalignments
  • Feature Importance Analysis — SHAP-powered charts showing which graduate attributes (GPA, internships, communication) statistically drive placement success
Forecasting & Prediction
  • Enrollment Yield Forecasts — Time-series models projecting future intake based on historical demographic patterns
  • Salary Trajectory Modelling — Predictive estimates of early-career earnings by major, region, and demand trend
  • Mobility Pathway Mapping — Probability analysis showing how underemployed graduates transition into better-matched roles over time
02
Student Enrollment Intelligence Dashboard
For admissions administrators and individual students
Institutional Analytics & Risk Tools
  • Enrollment Trend Forecasting — LSTM-driven projections of future enrollment volumes segmented by program and demographic group
  • Admission Risk Modelling — Interactive controls for administrators to simulate over- and under-enrollment scenarios at configurable risk thresholds (5%, 10%, 20%)
  • Placement Rate by Department — Comparative performance metrics across all university streams, enabling data-driven curriculum investment decisions
Student-Facing Recommender
  • Placement Probability Evaluator — Students input their profile (GPA, internship count, soft skills, projects) and receive a real-time employment likelihood score powered by XGBoost
  • Personalised Skill-Building Engine — AI-driven recommendation of specific certifications and technical skills (e.g. SAP, data analysis, cloud) to close the gap between a student's current profile and their target career
  • Alternative Pathway Suggestions — For students in lower-demand fields, the system recommends adjacent disciplines or regions where their profile performs better
03  —  System Architecture

How the System Is Structured

The platform follows a layered pipeline architecture — data enters at the top, is engineered into features, passes through multiple ML models, and exits as forecasts and recommendations that feed the dashboards. A feedback loop captures actuals to continuously retrain the models on new enrollment data.

Data Sources
Historical enrollment records Labor market & salary data Population & demographic trends GDP · tuition · grants
Feature Pipeline
Lag features Rolling averages Seasonality encoding Interaction terms Normalisation
ML Model Layer
LSTM / GRU — temporal patterns XGBoost — tabular data Prophet — trend & seasonality Ensemble stacking
Forecast Outputs
Demand forecast by program & year Market gap (supply vs. demand) Capacity & intake forecast Confidence bands
Recommendation
Enrollment recommendation logic ROI optimisation Fill rate maximisation Labor market alignment scoring
Monitoring
MAE · RMSE · MAPE tracking on actuals Scheduled retraining pipeline Feedback loop

05  —  Delivery Timeline

Phase-by-Phase Plan — 8 to 10 Weeks

Work is structured in seven sequential phases, with each phase building directly on the last. The sequence is designed so there is a working, demonstrable version of the system by Week 6 — the final weeks focus on integration, refinement, and delivery.

Weeks 1–2
Phase 1  ·  Scoping, Data & Architecture
Define data schema and source datasets (enrollment, job market, demographics). Set up project repository and database. Design API structure. Wireframe dashboard layouts for both modules. Confirm data availability and simulation strategy.
Weeks 3–4
Phase 2  ·  Data Pipeline & Feature Engineering
Build the ETL pipeline for ingesting and cleaning all data sources. Engineer lag features, rolling averages, seasonality encoding, and interaction terms. Store processed feature sets in the database. Unit test the pipeline end-to-end.
Weeks 5–6
Phase 3  ·  Machine Learning Model Development
Train and evaluate the XGBoost placement predictor, Prophet enrollment forecast, and LSTM temporal demand model. Build SHAP explainability layer. Assess performance using MAE, RMSE, and MAPE. Develop ensemble stacking to combine model outputs.
Week 6–7
Phase 4  ·  Backend API Development
Build FastAPI server exposing endpoints for all forecast, prediction, and recommendation functions. Wrap trained models into API services. Add input validation, error handling, and CORS configuration for frontend consumption.
Weeks 7–8
Phase 5  ·  Frontend Dashboard Development
Build React application covering both modules. Module 1: market demand charts, geographic maps, skill gap visuals, feature importance displays. Module 2: enrollment forecasts, admin risk controls, student placement evaluator, skill recommendations. Connect all views to the live API.
Weeks 8–9
Phase 6  ·  Recommendation Engine & Full Integration
Implement the enrollment recommendation logic (ROI optimisation, fill rate maximisation, labor market alignment). Build the personalised skill-building engine and pathway mapper. Complete end-to-end integration testing across all modules.
Weeks 9–10
Phase 7  ·  QA, Monitoring & Final Delivery
Full quality assurance pass across all features. Configure model performance monitoring and scheduled retraining pipeline. Prepare technical documentation, setup guide, and API reference. Deliver final working application with walkthrough.
